🩺 Blood Pressure Tracker Project Spec

A personal, mobile-friendly web application for logging blood pressure measurements from an iPhone, with CSV export for time-series analysis.

Goals

  • Primary: Enter and store blood pressure readings (systolic, diastolic, pulse, date/time)
  • Secondary: Export all measurements as a CSV file

Tech Stack

LayerTechnology
FrontendReact (mobile-first, responsive)
BackendNode.js + Express
DatabasePostgreSQL
HostingSynology NAS (Docker)
ClientiPhone (Safari browser)

Architecture

iPhone (Safari)
    ↓ HTTP over local network
Synology NAS
    ├── Docker: Node.js + Express  (API + serves React build)
    └── Docker: PostgreSQL         (persistent data volume)

Access via http://<NAS-local-IP>:3000 — no internet exposure needed.

Data Model — measurements table

ColumnTypeNotes
idSERIAL PKAuto-increment
measured_atTIMESTAMPTZWhen measurement was taken
systolicINTEGERmmHg — upper number
diastolicINTEGERmmHg — lower number
pulseINTEGERBPM — optional
notesTEXTOptional free text
created_atTIMESTAMPTZRecord creation time

API Endpoints

MethodEndpointDescription
POST/api/measurementsSave new reading
GET/api/measurementsRetrieve all readings (JSON)
GET/api/export/csvDownload all data as CSV

Frontend Screens

  1. Input Form — date/time (default: now), systolic, diastolic, pulse, notes. Mobile-optimized numeric inputs.
  2. History List — scrollable list of all past readings, newest first.
  3. Export Button — triggers CSV download via /api/export/csv.

Synology NAS Deployment

  1. Install Docker via Synology Package Center
  2. Create docker-compose.yml with postgres + app services
  3. PostgreSQL data in a named Docker volume (persistent across restarts)
  4. Expose port 3000 on the NAS

Coding Sequence

  1. npm init — initialize Node project
  2. Express server + API routes
  3. PostgreSQL connection via pg library
  4. DB schema / migration script
  5. React frontend: form + history list + export button
  6. npm run build — React production build served via Express
  7. Write docker-compose.yml for NAS
  8. End-to-end test from iPhone on local network

Future Enhancements

  • Authentication for local network privacy
  • Charts showing BP trends over time
  • Date range filtering in history view
  • Measurement reminders / notifications
